# Module 4 — ArgoCD for Automatic Deployment

⚠️ **Medium**

## What was built

- `argocd/application.yaml` — ArgoCD Application CR; watches `helm/chatbot/` on `main`, auto-syncs on every change
- Updated `.github/workflows/ci.yml` — stamps `image.tag` with the git SHA after every image push, commits back with `[skip ci]`
- Updated `helm/chatbot/values.yaml` — `pullPolicy: Always` → `IfNotPresent` (SHA tags are immutable)

## Architecture

```
Developer push to main
    │
    ▼
GitHub Actions
  1. Build + push :latest and :<sha> to Docker Hub
  2. sed values.yaml → image.tag: <sha>
  3. git commit "[skip ci]" + push
    │
    ▼  (ArgoCD polls Git every ~3 min)
ArgoCD (in-cluster, namespace: argocd)
  4. Detects values.yaml changed on main
  5. helm template + kubectl apply diff
  6. New pod starts with :<sha> image
    │
    ▼
http://localhost:8501
```

ArgoCD runs **inside** the local Docker Desktop cluster — GitHub Actions runners cannot reach it. So CI writes to Git instead of calling `argocd app sync`. ArgoCD picks it up on its next poll (default: 3 min).

`selfHeal: true` reverts any manual `kubectl` changes back to what Git says within one polling cycle.

## Install ArgoCD (one-time setup)

```powershell
# 0 — uninstall existing manual Helm release first (avoid resource conflicts)
helm.exe uninstall chatbot   # "not found" is OK

# 1 — install
kubectl create namespace argocd
kubectl apply -n argocd -f https://raw.githubusercontent.com/argoproj/argo-cd/stable/manifests/install.yaml
kubectl wait --for=condition=Ready pods --all -n argocd --timeout=300s

# 2 — access UI (run in a dedicated terminal, keep it running)
kubectl port-forward svc/argocd-server -n argocd 8080:443
# UI: https://localhost:8080

# 3 — get admin password
$b64 = kubectl get secret argocd-initial-admin-secret -n argocd -o jsonpath="{.data.password}"
[System.Text.Encoding]::UTF8.GetString([System.Convert]::FromBase64String($b64))

# 4 — apply the Application
kubectl apply -f argocd/application.yaml

# 5 — watch sync (or check UI)
kubectl get pods   # chatbot-chatbot-<hash>  1/1  Running
```

## Day-to-day commands

```powershell
# CLI (optional — download argocd-windows-amd64.exe, put on PATH)
argocd.exe login localhost:8080 --username admin --insecure
argocd.exe app get chatbot      # sync + health status
argocd.exe app sync chatbot     # force sync without waiting for poll
argocd.exe app diff chatbot     # diff: Git vs cluster
```
