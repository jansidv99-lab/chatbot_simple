# Module 3 — Helm Charts + Kubernetes Deployment

⚠️ **Medium**

## What was built
- `helm/chatbot/Chart.yaml` — chart metadata
- `helm/chatbot/values.yaml` — image, service, ollama config
- `helm/chatbot/templates/deployment.yaml` — single-replica Streamlit deployment
- `helm/chatbot/templates/service.yaml` — LoadBalancer on port 8501

## Architecture
- `LoadBalancer` service → Docker Desktop assigns `localhost` as external IP automatically
- `host.docker.internal` used as Ollama host — resolves to Windows host from inside pods
- `OLLAMA_HOST` and `MODEL_NAME` injected via env vars (not ConfigMap — small number of vars)
- Liveness probe on `/_stcore/health` (Streamlit built-in endpoint)
- `pullPolicy: Always` ensures `:latest` tag is always pulled fresh

## Before deploying — edit values.yaml
Replace `<DOCKERHUB_USERNAME>` in `helm/chatbot/values.yaml` with your Docker Hub username.

## Deploy commands
```bash
# Lint
helm.exe lint helm/chatbot

# Preview manifests
helm.exe install chatbot helm/chatbot --dry-run --debug

# Deploy
helm.exe install chatbot helm/chatbot

# Upgrade after changes
helm.exe upgrade chatbot helm/chatbot

# Uninstall
helm.exe uninstall chatbot
```

## Verify
```bash
kubectl get pods                          # chatbot pod → Running
kubectl get svc                           # EXTERNAL-IP = localhost
kubectl logs deployment/chatbot-chatbot   # check for errors
```
Then open http://localhost:8501
