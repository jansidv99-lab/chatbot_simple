## Schema

```sql
CREATE TABLE IF NOT EXISTS daily_positions (
    symbol                VARCHAR    NOT NULL,
    trade_date            DATE       NOT NULL,
    segment               VARCHAR,
    position_type         VARCHAR,
    open_quantity         NUMERIC,
    open_average          NUMERIC,
    open_value            NUMERIC,
    previous_close_price  NUMERIC,
    closing_value         NUMERIC,
    unrealized_profit     NUMERIC,
    unrealized_profit_pct NUMERIC,
    PRIMARY KEY (trade_date, symbol)
);
```

Source: `raw_data_files/daily_poistions/positions-19-5.xlsx`, sheet `F&O`, headers row 15, data from row 16. `client_id` (cell C7) never read or stored.

## Key design decisions

- `read_only=True` breaks `openpyxl.iter_rows(min_row=N)` — parser uses default (non-read-only) mode
- `ON CONFLICT (trade_date, symbol) DO NOTHING` handles duplicates at DB level; rowcount 0/1 per row gives accurate inserted/skipped counts
- Postgres service name is fixed (`postgres`, not release-prefixed) so `PG_HOST=postgres` resolves without Helm templating in the app
- `emptyDir` volume — data is ephemeral; acceptable for Phase 2 dev; add PVC later

## Known limitations

- Postgres data is ephemeral (`emptyDir`) — pod restart loses data
- Password is plaintext in `values.yaml` — refactor to K8s Secret later
- `daily_pl` table (for `raw_data_files/daily_pl/`) not yet implemented
