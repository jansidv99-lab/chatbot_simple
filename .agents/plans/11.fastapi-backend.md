## Architecture

```
Streamlit pages  →  FastAPI (http://localhost:8000)  →  LangGraph / Ollama / PostgreSQL
```

All Streamlit pages that call the LLM or agent pipeline now go through FastAPI, giving a single HTTP contract with JWT enforcement and structured error handling.

## Endpoints

| Method | Path | Auth | Purpose |
|---|---|---|---|
| `POST` | `/auth/register` | No | Create account → `{user_id, username}` |
| `POST` | `/auth/login` | No (OAuth2 form) | Authenticate → `{access_token, refresh_token, token_type}` |
| `POST` | `/chat/` | Bearer JWT | Stream chat tokens as SSE (`text/event-stream`) |
| `POST` | `/analytics/` | Bearer JWT | Run LangGraph pipeline → JSON result |
| `GET` | `/health` | No | `{"status": "ok"}` |
| `GET` | `/docs` | No | Swagger UI |

## Key Design Choices

- Route handlers use `def` (not `async def`) — LangGraph and psycopg2 are both sync; using `def` lets Uvicorn run them in a thread pool without blocking the event loop.
- `api/deps.py` `get_current_user()` re-raises `401` for `ExpiredSignatureError`, `InvalidTokenError`, or wrong token type — all other errors fall to the global 500 handler.
- `chat.py` SSE format: each token is emitted as `data: <token>\n\n`. Streamlit strips the `data: ` prefix before yielding to `st.write_stream`.
- `analytics.py` does a `json.dumps(default=str)` round-trip to coerce `date` objects in `query_results` to strings before serialisation.
- `pages/login.py` still calls `auth.users` and `auth.tokens` directly (not through FastAPI) — login page creates JWT tokens locally and stores them in `st.session_state`.
- `pages/upload.py` still calls `ingestion.db` / `ingestion.parser` directly — ingestion is not exposed via FastAPI.

## SSE Consumption Pattern (Streamlit → FastAPI)

```python
# app.py
with httpx.Client(timeout=120) as client:
    with client.stream("POST", f"{API_BASE_URL}/chat/", json=...,
                       headers={"Authorization": f"Bearer {token}"}) as resp:
        if resp.status_code != 200:
            raise ConnectionError(...)
        for line in resp.iter_lines():
            if line.startswith("data: "):
                yield line[6:]
```

## New Env Var

| Variable | Default | Purpose |
|---|---|---|
| `API_BASE_URL` | `http://localhost:8000` | FastAPI base URL used by Streamlit pages |

## Startup Order (Local Dev)

FastAPI must start **before** Streamlit — Streamlit calls `API_BASE_URL` on every chat/analytics request.

```powershell
# Terminal 1
uvicorn api.main:app --reload --port 8000

# Terminal 2
streamlit run app.py
```

## Known Limitations

- `api/` is not in the `Dockerfile` — the Docker/k8s image only runs Streamlit; FastAPI is local-dev only post-Module 11. Adding `api/` to the Dockerfile and deploying it as a separate service (or sidecar) is deferred to a future module.
- `api/` is excluded from CI lint scope (matches `agents/` exclusion — ruff only checks `app.py pages/ ingestion/`).
- No FastAPI integration tests yet — only unit tests for individual utilities.
