# Module 2 — GitHub + CI Pipeline

✅ **Simple**

## What was built
- `Dockerfile` — containerizes the Streamlit app on `python:3.11-slim`
- `.dockerignore` — excludes `.env`, caches, docs from the image
- `.github/workflows/ci.yml` — GitHub Actions workflow: builds and pushes Docker image to Docker Hub on every push to `main`
- Git repo initialized and committed

## Architecture
- Image entry point: `streamlit run app.py --server.address=0.0.0.0 --server.headless=true`
- `OLLAMA_HOST` and `MODEL_NAME` are NOT baked into the image — injected at runtime via env vars
- CI uses official Docker actions: `docker/login-action@v3` + `docker/build-push-action@v6`
- Two tags pushed per build: `:latest` and `:<git-sha>` for traceability

## GitHub Secrets required
| Secret | Purpose |
|---|---|
| `DOCKERHUB_USERNAME` | Docker Hub login |
| `DOCKERHUB_TOKEN` | Docker Hub access token (not password) |

Generate token at: hub.docker.com → Account Settings → Security → New Access Token

## Local Docker test
```bash
docker build -t chatbot-simple .
docker run -e OLLAMA_HOST=http://host.docker.internal:11434 -p 8501:8501 chatbot-simple
```
